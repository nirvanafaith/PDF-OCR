# 精修（Refine）模块文档

## 一、整体定位与数据流

### 1.1 在流程中的位置

精修是整个校对流程的**最终阶段**（第四阶段），接在横校之后。由 `MainWindow._setup_refine_stage` 创建，直接引用横校窗口的 `page_lines` 属性（`self.horiz_widget.page_lines`），因此横校页面中用户所做的所有文本修改（包括 `_sync_chars_with_text` 更新的字符级数据）均直接传递到精修。**精修阶段完成后**，输出 PDF 文件并结束整个流程，回到导入阶段。

### 1.2 输入数据规格

| 参数 | 类型 | 说明 |
|------|------|------|
| `page_lines` | `dict[int, list[LineSlice]]` | 同横校的输入，但 text 和 chars 可能已被横校阶段修改 |
| `page_images` | `list[PIL.Image]` | 同上，300 DPI 渲染的 PDF 页面图像 |

### 1.3 数据转换流程

`_convert_chars` 方法将 `page_lines` 中的字符数据转换为精修使用的 `RefineTextItem`：

- 遍历每个页面、每行，跳过 `_ignored` 为 True 的行。
- 对行内每个字符数据，提取 `text`、`bbox`，计算 `font_size = bbox_height`。
- 创建 `RefineTextItem`，按页码归类存入 `self.page_items`。
- 这种 **逐字符独立化** 的设计使得精修阶段不再有"行"的概念——每个字符完全独立，可以被单独拖拽、缩放、修改或删除。

### 1.4 输出数据规格

`_build_corrected_chars` 方法构建 `list[CorrectedChar]`，包含：`text`（最终校对文本）、`bbox`（用户调整后的边界框）、`page_num`、`ignored`。此数据传递给 `PDFOutputGenerator.generate` 生成输出 PDF。

## 二、详细功能分解

### 2.1 字符框组件（MovableTextItem）架构

`MovableTextItem` 继承自 `QGraphicsRectItem`，是精修的核心交互单元。每个 `MovableTextItem` 对应一个 `RefineTextItem` 数据模型。

#### 2.1.1 层级结构

```
MovableTextItem (QGraphicsRectItem)
├── 矩形本身 (透明背景，选中时显示蓝色虚线边框)
├── _text_item (QGraphicsTextItem, 子图元)
│   └── 红色文字 (Qt.GlobalColor.red)，微软雅黑字体
└── _handles (dict[str, QGraphicsRectItem]) 共8个
    ├── topLeft, top, topRight   (上边三手柄)
    ├── left, right               (左右两手柄)
    └── bottomLeft, bottom, bottomRight (下边三手柄)
```

#### 2.1.2 交互状态机

- **未激活状态**（默认/手型模式）：不响应鼠标事件，无悬停反馈，不可选中。
- **激活状态**（拖拽模式）：`_activated = True`，启用 `setAcceptHoverEvents(True)`。
  - **空闲**：悬停时根据位置显示缩放或移动光标。
  - **移动中**（`_moving = True`）：鼠标左键按下时进入，`mouseMoveEvent` 手动更新 `setPos`，`mouseReleaseEvent` 退出。移动过程中同步更新缩放手柄位置。
  - **缩放中**（`_resize_handle != None`）：点击到缩放手柄时进入，`mouseMoveEvent` 根据手柄类型（8方向）调整矩形宽度、高度和位置。当矩形宽或高小于 5px 时钳制。缩放时同步更新字号、手柄位置和文字居中。
  - **选中**（`_selected = True`）：显示蓝色虚线边框和蓝色缩放手柄。

#### 2.1.3 视觉反馈

| 状态 | 矩形边框 | 手柄 | 文字颜色 | 光标 |
|------|---------|------|---------|------|
| 未选中 (空闲) | 无边框 | 隐藏 | 红色 | 默认箭头 |
| 选中 | 蓝色虚线 (1px) | 蓝色方块 (8x8px) | 红色 | 取决于位置 |
| 手柄上 (悬停) | 蓝色虚线 | 可见 | 红色 | 8方向缩放光标 |
| 文本区 (悬停) | 蓝色虚线 | 可见 | 红色 | 移动光标 (SizeAllCursor) |

#### 2.1.4 右键菜单

`contextMenuEvent` 弹出两个选项：
- **修改文字**：弹出对话框，`QLineEdit` 预填当前文本，确认后更新 `_data.text` 和 `_text_item` 显示。
- **删除**：设置 `_data.ignored = True`，`setVisible(False)` 隐藏图元。

### 2.2 三种工作模式详解

#### 2.2.1 手型工具模式（`_on_hand_tool_toggle`）

- **互斥逻辑**：激活时强制关闭拖拽和新增文字模式（设 `_drag_mode = False`、`_add_text_mode = False`），取消另外两个按钮的选中状态。
- **行为**：遍历场景所有 `MovableTextItem` 调用 `deactivate()`，视图设为 `ScrollHandDrag`，光标为 `OpenHandCursor`。
- **取消时**：恢复 `NoDrag` 和 `ArrowCursor`。

#### 2.2.2 拖拽模式（`_on_drag_toggle`）

- **互斥逻辑**：激活时关闭手型和新增文字模式。
- **行为**：遍历场景所有 `MovableTextItem` 调用 `activate()`，视图设为 `NoDrag`，光标 `ArrowCursor`。
- **取消时**：调用所有 `MovableTextItem.deactivate()`，清除选中项引用。
- **点击空白取消选中**：`eventFilter` 监听鼠标左键按下，若点击位置不在任何 `MovableTextItem` 上，则遍历场景取消所有选中状态。
- **Delete 键删除**：`keyPressEvent` 处理 `Qt.Key_Delete`，找到当前选中项标记为忽略并隐藏。

#### 2.2.3 新增文字模式（`_on_add_text_toggle`）

- **互斥逻辑**：激活时关闭手型和拖拽模式。
- **行为**：视图设为 `NoDrag`，光标 `CrossCursor`，停用所有文字项。
- **右键添加**：`_on_context_menu` 检测到新增文字模式下且点击位置不在已有文字项上时，提供"添加文字"菜单项。
- **添加流程**（`_add_text_at`）：
  1. 弹出对话框输入文字内容。
  2. 计算平均字号（`_get_avg_font_size`，遍历当前页未忽略项的 `font_size` 求平均，默认 20.0）。
  3. 按场景坐标 / zoom_level 反算原始坐标下的插入点 `(base_x, base_y)`。
  4. 每个字符宽度和高度均设为平均字号，依次向右排列（`char_x = base_x + i * w`）。
  5. 为每个字符创建独立的 `RefineTextItem` 和 `MovableTextItem`，若当前处于拖拽模式则激活。

### 2.3 数据同步机制（`_sync_current_page`）

这是一个**关键的设计模式**——在任何可能改变场景内容或进入新页面的操作前调用：

- 遍历场景中所有 `MovableTextItem`。
- 从 `item.pos()` 和 `item.rect()` 获取场景坐标系下的位置和尺寸。
- 除以 `item._zoom_level` 反算原始坐标系下的 bbox：`[pos.x/z, pos.y/z, (pos.x+w)/z, (pos.y+h)/z]`。
- 更新 `data.text = item._text_item.toPlainText()`（支持就地编辑下的文本变更）。
- 更新 `data.font_size = rect.height() / z`。

此方法在以下时机被调用：翻页、页码跳转、缩放（缩放输入/放大缩小/适合宽度高度）、输出按钮。**确保了用户的拖拽和缩放操作在任何界面切换前被持久化到数据模型**。

### 2.4 PDF 输出生成

#### 2.4.1 双版本输出

精修每次输出都生成两个 PDF 文件：

- **红色文字版**（`_红.pdf`）：文字以 RGB(255,0,0) 渲染，适合人工审阅。
- **透明文字版**（`_透明.pdf`）：文字以 `RGBA(0,0,0,alpha=0)` 渲染（完全透明），适用于需要文字区域占位但不可见的场景（如出版排版中的预留空间）。

#### 2.4.2 生成流程

1. `_build_corrected_chars`：按页码顺序遍历所有 `RefineTextItem`，生成 `list[CorrectedChar]`。
2. 用户选择保存路径（`QFileDialog.getSaveFileName`）。
3. `_start_pdf_generation` 启动工作线程：
   - 创建 `PDFOutputGenerator` 实例和 `PDFOutputWorker`（继承 `QThread`）线程。
   - 显示模态进度条（`QProgressDialog`，范围 0-100，无取消按钮，不自动关闭）。
   - 信号连接：`progress_signal` → 更新进度条百分比和描述文字；`finished_signal` → 关闭进度条、弹出成功消息（包含两个文件路径）、若为确认完成操作则发射 `finished_signal`；`error_signal` → 关闭进度条、弹出错误对话框。
4. `PDFOutputGenerator.generate` 输出版本：
   - 以 `reportlab` 为底层 PDF 生成库。
   - 每页以原始页面图像为底图（`c.drawImage(ImageReader(img), ...)`）。
   - 逐个字符绘制：计算字号为 `bbox_height`，若字符串渲染宽度超出 bbox 宽度则按比例缩小字号，水平居中绘制。
   - 使用微软雅黑字体（`MicrosoftYaHei`，从 `C:\Windows\Fonts\msyh.ttc` 注册），回退为 Helvetica。

#### 2.4.3 两种输出入口对比

| 入口 | 行为 | 完成后 |
|------|------|--------|
| **输出按钮**（`_on_output`） | 生成两份 PDF，完成后弹出成功提示 | 停留在精修界面 |
| **确认完成按钮**（`_on_finish_confirm`） | 先确认对话框，再保存对话框，生成两份 PDF | 完成后发射 `finished_signal`，主窗口清理所有窗口，回到导入阶段 |

### 2.5 缓存与性能优化

- **像素缓存**（`_pixmap_cache`）：单条目缓存，键为 `(current_page, zoom_level)`，切换页面或缩放时仅保留当前组合，自动淘汰旧缓存。
- **平均字号计算**：每次新增文字时遍历当前页所有未忽略项，非缓存结果（页面数据量通常较小）。
- **页面渲染**：每次 `_render_page` 清空场景重建所有图元，无增量更新机制。

### 2.6 返回机制

`back_signal` 发射后，主窗口销毁精修窗口，恢复横校窗口显示。精修阶段**不保留退出状态**，返回后所有精修阶段的修改（拖拽位置、缩放、新增、删除）将丢失，用户需重新进入精修。

## 三、详细 UI 布局

### 3.1 顶部工具栏

从左到右：

```
| ← 返回 | 分隔线 | 上一页 | 第N/M页 | [页码输入] | 下一页 | 分隔线 | 放大 | [100%] | 缩小 | 适合宽度 | 适合高度 | 分隔线 | 手型工具 | 拖拽 | 新增文字 | 分隔线 | 输出 | 确认完成(绿色) |
```

- **手型工具/拖拽/新增文字**：三个 `QPushButton`，均为 `setCheckable(True)`，互斥逻辑在各自事件处理中手动维护。
- **确认完成按钮**：独有样式——绿色背景（`#198754`），白色加粗文字，圆角 4px，悬停时变深绿（`#157347`）。

### 3.2 中部主区域（单图形视图）

区别于横校的双视图，精修采用**单视图**布局，页面图像作为背景 `QGraphicsPixmapItem`（Z 值 0），文字项在 Z 值 1 的前景层。

- 场景背景白色。
- 视图设置：抗锯齿渲染、`SmartViewportUpdate`、`CustomContextMenu`。
- 事件过滤器绑定到 `view.viewport()`，处理 Ctrl+滚轮缩放和边界滚轮翻页。

### 3.3 模态对话框

**修改文字对话框**（同横校类似）：
- 标题"修改文字"，最小宽 400px。
- `QLineEdit` 预填 + 全选，回车确认。
- 确认后更新数据模型和界面显示。

**添加文字对话框**：
- 标题"添加文字"，最小宽 300px。
- 仅一个文本输入框 + 确定/取消按钮。
- 无预设文本，输入为空则取消。

**确认完成对话框**：
- `QMessageBox.question` 风格，标题"确认完成"。
- 提示："确认完成后将保存精修结果并返回画框页面。是否确认？"
- Yes/No 按钮。

**PDF 生成进度条**：
- `QProgressDialog`，标题"正在生成PDF"。
- 模态（`WindowModal`），无取消按钮，不自动关闭/重置。
- 进度描述动态变化："正在生成红色文字版... (第 N/M 页)" → "正在生成透明文字版... (第 N/M 页)"。
- 生成成功后自动关闭。

## 四、横校与精修的交互差异总结

| 对比维度 | 横校 | 精修 |
|---------|------|------|
| 操作粒度 | 行级 | 字符级 |
| 视图布局 | 左右双视图（文字叠加 + 原始 PDF） | 单视图（文字在背景图上） |
| 右键菜单 | 修改文字 / 忽略/删除 | 修改文字 / 删除（拖拽模式）；添加文字（新增文字模式） |
| 拖拽调整 | 不支持 | 支持（位置拖拽 + 四角缩放 + 8 方向手柄） |
| 文字新增 | 不支持 | 支持（新增文字模式 + 右键添加） |
| 键盘快捷键 | 无 | Delete 键删除选中项 |
| 数据持久化 | 修改即时写入 LineSlice | 修改先写入场景，翻页/缩放时同步到数据模型 |
| 输出格式 | 无直接输出（数据传给精修） | 双版本 PDF（红色 + 透明） |
| 字体颜色 | 黑色/灰色（忽略） | 红色 |
| 行预览 | 鼠标悬停时行切片图像预览 + 右侧蓝色框高亮 | 无预览功能 |
| 页面同步 | 左右视图滚动联动 | 无联动需求 |
