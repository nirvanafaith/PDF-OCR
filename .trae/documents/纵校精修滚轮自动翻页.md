# 纵校和精修窗口滚轮自动翻页功能

## 技术报告

### 需求

纵校和精修窗口的PDF阅读界面，当滚轮让页面向上滚到顶了就向前翻页，滚轮让页面向下滚到底了就向后翻页。

### 现状分析

| 窗口 | 滚轮翻页 | Ctrl+滚轮缩放 | 翻页前同步数据 |
|------|---------|-------------|-------------|
| 画框窗口 | **已实现** | 已实现 | 不需要 |
| 纵校窗口 | **未实现** | 已实现 | 不需要 |
| 精修窗口 | **未实现** | 已实现 | **需要** `_sync_current_page()` |

画框窗口已在 `eventFilter` 中实现了滚轮自动翻页逻辑（[draw_box_window.py:338-355](file:///c:/Users/E-VR/Documents/trae_projects/横校/ui/draw_box_window.py#L338-L355)），纵校和精修窗口缺少此功能。

### 实现方案

参考画框窗口的滚轮翻页逻辑，在纵校和精修窗口的 `eventFilter` 中新增 `QWheelEvent` 处理。

#### 核心算法

```
当滚轮向上滚动(delta > 0) 且 滚动条在顶部(value == minimum):
    → 翻到上一页，渲染后滚动条跳到底部(maximum)
当滚轮向下滚动(delta < 0) 且 滚动条在底部(value == maximum):
    → 翻到下一页，渲染后滚动条跳到顶部(minimum)
边界处拦截事件(return True)，非边界放行(return False)
```

使用 `QTimer.singleShot(0, lambda: ...)` 延迟设置滚动条位置，确保 `_render_page()` 完成场景重建后再操作滚动条。

#### 修改 1：纵校窗口 (vertical_check_window.py)

在 `eventFilter` 方法中，Ctrl+滚轮缩放处理之后、鼠标悬停处理之前，插入滚轮翻页逻辑。

- **位置**：第 272 行之后
- **无需** `_sync_current_page()`（纵校只修改文本，不涉及位置变更）
- QTimer 已导入

#### 修改 2：精修窗口 (refine_window.py)

在 `eventFilter` 方法中，Ctrl+滚轮缩放处理之后、鼠标点击处理之前，插入滚轮翻页逻辑。

- **位置**：第 832 行之后
- **必须** 调用 `_sync_current_page()`（精修涉及文字位置拖拽修改，翻页前必须同步数据）
- QTimer 已导入

### 边界情况处理

1. **页面内容等于视口大小**（minimum == maximum == value）：滚轮直接翻页 ✓
2. **Ctrl+滚轮**：缩放优先级高于翻页，`calculate_wheel_zoom` 返回非 None 时已拦截 ✓
3. **首页向上翻/末页向下翻**：条件判断 `current_page > 0` / `current_page < total_pages - 1` 阻止越界 ✓
4. **非边界滚轮**：不拦截事件，传递给 QGraphicsView 默认滚动行为 ✓
