# 纵校修改传递到精修 + 新增多字符输入

## 技术报告

### 问题1：纵校修改文字未传递到精修

#### 根本原因

数据流断裂：
1. 纵校 `_on_modify_text` 修改 `LineSlice.text`（行级文本），但**未同步更新** `LineSlice.chars`（字符级列表）
2. 精修 `_convert_chars()` 读取 `LineSlice.chars`（字符级），而非 `LineSlice.text`（行级）
3. 因此纵校中的文字修改不会出现在精修中

#### 修复方案

在纵校 `_on_modify_text` 中，修改 `ls.text` 后同步更新 `ls.chars`：

- **字符数相同**：逐个替换原字符的 `text`，保留原 `bbox`（最精确）
- **字符数不同**：按行 bbox 等间距分配新字符的 bbox（近似但合理）

**修改文件**：`ui/vertical_check_window.py`

1. 新增方法 `_sync_chars_with_text(self, ls)`
2. 在 `_on_modify_text` 中调用

### 问题2：新增文字只允许输入一个字符

#### 现状

`_add_text_at` 中 `line_edit.setMaxLength(1)` 限制只能输入1个字符。

#### 修复方案

- 移除 `setMaxLength(1)` 限制
- 为输入的每个字符创建独立的 `RefineTextItem` + `MovableTextItem`
- 第一个字在输入位置，后续字按序向右偏移一个字宽

**修改文件**：`ui/refine_window.py`

1. 修改 `_add_text_at` 方法

### 验证步骤

1. `py_compile` 编译验证
2. VS Code 诊断
3. 手动测试：纵校修改文字 → 精修查看是否已更新
4. 手动测试：精修新增多字符 → 查看是否按序排列
