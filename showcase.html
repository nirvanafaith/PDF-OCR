PDF-OCR

可视化字符级 PDF OCR 精修工具。把 OCR 识别结果与原始扫描图像紧密结合， 让用户在视觉上下文中逐字符地校对、修改、删除、补全，最终输出经过人工精修的高质量 PDF 文本层。

01 · 项目简介

不只是 OCR，更是字符级精修工作流

传统 OCR 工具只输出整段文本，要么全盘接受，要么在文本编辑器里逐字核对——效率低且容易出错。 PDF-OCR 提供完整的「画框选区 → 字符级 OCR → 纵校（按字符分组）→ 横校（按行校对）→ 精修输出」可视化工作流， 让校对回到原文上下文中进行。

02 · 痛点与解决方案

六个痛点，六种解法

扫描书页 PDF 的字符级 OCR 长期被忽视。我们针对每个痛点设计了对应功能， 让校对工作真正回到「看得见、改得动」的状态。

❌ 只给文本，不可精修
OCR 引擎输出一段识别文本，复制到编辑器里逐字核对，看不到原文上下文，错字难以发现。
✅ 四阶段流水线 + 精修窗口
导入 → 纵校 → 横校 → 精修，每个阶段都有可视化交互。精修窗口支持拖拽、8 手柄缩放、右键删除。
❌ 行级识别，丢失字符信息
多数 OCR 以行为最小单位，单字置信度与字符边界框被丢弃，错字无法定位。
✅ 字符级 OCR
每个字符独立保存 char_id、line_id、page_num、char、score、box，可定位到「第 3 页第 7 行第 4 个字」粒度。
❌ 跨页同字错误难修正
同一个常用字全文出现上千次，OCR 系统性偏差（如「己」识别成「已」）需在每页单独修正。
✅ 纵校模式（按字符分组）
把全文中同一个字聚合到一起，用户一次性校对所有实例，修改一次全文同步更新。
❌ 大 PDF 翻页卡顿
扫描书页 PDF 动辄数百页，全量渲染撑爆内存，翻页延迟严重。
✅ LazyPageLoader 懒加载
按需渲染 + LRU 缓存，最近访问的页面保留内存，超过阈值自动淘汰。
❌ 缺乏视觉反馈
识别结果与原图脱离，用户无法直观看到「OCR 在哪个区域识别错了」。
✅ 横校模式（按行校对）
按页逐行展示识别文本叠加在原图上，错误一目了然。鼠标悬停高亮，双击修改。
❌ 单版本职责臃肿
画框 OCR 推理与校对精修混在同一份代码中，模块膨胀、难以并行演进。
✅ Python 双版本
software1（PyQt6 + RapidOCR）承担画框 + OCR 推理，software2（PyQt6 + PyMuPDF）承担校对 + 精修输出。
03 · 核心亮点

八大核心功能

从画框选区到 GPU 加速，从纵校分组到懒加载翻页，每一项功能都对应一个真实痛点。

🖼️
3.1 可视化画框选区
用户在 PDF 页面上画框选区，OCR 只处理选区内内容，避免页眉、页脚、水印、批注等无关内容干扰识别结果。 让 OCR 聚焦在真正需要识别的正文区域，准确率与速度都显著提升。
🔤
3.2 字符级 OCR
不仅识别文本行，还能识别单个字符并保存字符级元数据（char_id / line_id / page_num / char / score / box）。 可定位到「第 3 页第 7 行第 4 个字」粒度进行精修。
📊
3.3 纵校模式（按字符分组）
把全文中同一个字符的所有出现聚合到一起，用户一次性校对同字所有实例。 例如「己」字被系统性识别为「已」，纵校模式下选择「己」字符组，修改一次全文同步更新。
📖
3.4 横校模式（按行校对）
按页逐行展示识别文本与原图叠加视图，错误一目了然。鼠标悬停高亮当前行，双击修改文字， 右键忽略该行，Ctrl + 滚轮缩放。pixmap 缓存避免重复缩放。
✏️
3.5 精修窗口
拖拽移动文本块位置，8 个手柄（四角 + 四边）调整大小，右键删除不需要的实例。 导出双 PDF：红色文字版（人工核对）+ 透明文字版（机器后续处理，可作可搜索 PDF 文本层）。
🔀
3.6 双版本实现
software1（Python）：含完整 OCR 推理（RapidOCR + PaddleOCR + Surya）
software2（Python）：专注数据处理，OCR 结果以 JSON 导入
⚡
3.7 GPU 加速
通过 ONNX Runtime 后端切换 GPU/CPU 执行，自动处理 cuDNN 路径。 支持批量 OCR 识别提升 GPU 利用率，数百页 PDF 推理从数十分钟缩短至数分钟。
🌀
3.8 懒加载与流畅翻页
LazyPageLoader 按 (page, zoom) 缓存渲染后页面图像，大 PDF 也能流畅翻页。
04 · 操作流程

四阶段可视化工作流

每个阶段对应独立窗口，主窗口通过 QStackedWidget 按阶段切换，阶段间通过 Qt 信号传递数据， 支持阶段间回退——发现上游问题时随时返回修正。

1 导入阶段（ImportWindow）
选择 PDF 文件（software2 还支持直接导入已识别的 JSON 结果）。 后台线程 ImportWorker（worker object pattern）异步渲染 PDF 为页面图像，主线程不卡顿。 完成后通过 finished_signal 把 page_images、ocr_results、char_slices 传递给主窗口。
2 纵校阶段（VerticalCheckWindow）
左侧字符列表按字符聚合显示（按出现频率排序），点击某字符后中间预览区显示该字符全文所有出现的原图切片。 发现 OCR 错误时在右侧修改区修改文字（可改为多字符，如把「的」改为「目的」），所有实例同步更新。
3 横校阶段（HorizontalCheckWindow）
主视图按页显示原图 + 识别文字叠加。鼠标悬停高亮当前行，双击打开「修改文字」对话框， 右键菜单选择「忽略该行」或「修改文字」。工具栏切换页面、调整缩放。
4 精修阶段（RefineWindow）
显示横校输出的行文本块。鼠标拖拽调整位置，选中文本块出现 8 个手柄（四角 + 四边）调整大小， 右键删除不需要的实例。点击「导出 PDF」生成两份文件： 红色文字版（人工核对）+ 透明文字版（机器后续处理）。

💡 任何阶段都可通过「返回」按钮回到上一阶段（携带原数据重建上游窗口）

05 · 技术架构

主窗口 + 阶段窗口架构

两个 Python 版本均采用 MainWindow 持有 QStackedWidget 按阶段切换子窗口的架构。 阶段窗口之间通过 Qt 信号传递数据，窗口间无直接引用（低耦合）。

ImportWindow · 导入窗口
PDF → 页面图像（异步线程渲染）
↓
page_images, ocr_results, char_slices
VerticalCheckWindow · 纵校窗口
按字符分组校对，支持多字符修改
↓
updated char_slices, updated ocr_results
HorizontalCheckWindow · 横校窗口
按行校对，原图 + 文字叠加
↓
corrected_lines
RefineWindow · 精修窗口
拖拽 / 8 手柄缩放 / 右键删除
↓
输出

双版本技术栈

版本 目录 技术栈 角色
software1 software1/ PyQt6 + RapidOCR + PaddleOCR (PP-OCRv5) + PyMuPDF + Surya 基础版，含完整 OCR 推理（含画框选区）
software2 software2/ PyQt6 + PyMuPDF + reportlab 增强版，专注数据处理（导入 → 纵校 → 横校 → 精修），不做 OCR 推理

数据持久化（满足 BCNF）

项目使用 JSON 文件作为持久化存储，核心数据文件为 lines.json（行记录）与 chars.json（字符记录）。经评估，两者均满足 BC 范式（BCNF）——无传递依赖、部分依赖，所有非主属性完全依赖于候选键。

// lines.json 候选键：(page_num, line_id) { "line_id": 0, "page_num": 0, "text": "智慧工地技术", "score": 0.99988, "box": [[529.53, 437.16], [991.18, 437.16], [991.18, 521.64], [529.53, 521.64]] } // chars.json 候选键：(page_num, line_id, char_id) { "char_id": 0, "line_id": 0, "page_num": 0, "char": "智", "score": 0.99, "box": [x1, y1, x2, y2] }

08 · 未来规划

短期与长期路线图

从多语言支持到 AI 自动校对，从持久化到协同校对，PDF-OCR 还有更广阔的演进空间。

短期（3-6 个月） SHORT TERM
  • 多语言 OCR 支持：扩展日文、韩文、英文等更多语言的字符级 OCR
  • 校对结果持久化：引入 SQLite 存储纵校/横校修改，避免关闭程序丢失
  • 代码重复抽取：将 software1 与 software2 重复的 pdf_loader/styles/zoom_utils 抽取为公共子模块
  • 测试体系完善：测试脚本改为命令行参数，补充多页文档回归测试
长期（6-12 个月） LONG TERM
  • AI 自动校对：基于大语言模型（如接入 TRAE 自身 AI 能力）自动检测 OCR 可疑错误
  • 协同校对：支持多人同时校对同一文档，按页或按字符分配
  • Web 版：基于 WASM + Qt for WebAssembly 或纯 Web 重写，浏览器即可使用
  • 插件化 OCR 引擎：支持 RapidOCR、PaddleOCR、Surya、MinerU 等多后端热切换
  • 校对质量评估：基于置信度、修改频率等指标，自动生成质量报告，标注高风险区域
